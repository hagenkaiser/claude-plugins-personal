# Create Content — Interactive Session

Generic content creation workflow using the content-creator plugin scripts.

## Workflow

### Step 1: Determine what to create

Ask the user:
- Content type: single image, music-only reel, or narrated reel
- Platform: Instagram (free/public) or paid tier (members-only)
- Any creative direction, story beat, or brief to work from

Check if the project has a content plan (e.g. `data/generated/plans/week_N.json`) and use it if available.

### Step 2: Generate images

Use `generate_image.py` for SFW content. For each shot:
- Use the project's face reference image (`--face-ref`)
- Use `4:5` for portraits, `16:9` for landscapes, `9:16` for Reels
- Show each generated image to the user and get approval before continuing

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate_image.py \
  --prompt "scene description" \
  --output data/generated/shot1.jpg \
  --face-ref <project_face_ref> \
  --aspect 4:5
```

For NSFW content, use `generate_nsfw_image.py` instead — see NSFW section below.

### Step 3: Compose reel (if making a reel)

Every reel uses 3 shots that tell a mini story:
- Shot 1: wide/establishing
- Shot 2: mid/action
- Shot 3: close/intimate detail

**Music-only reel:**
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate_slideshow.py \
  --images shot1.jpg shot2.jpg shot3.jpg \
  --music <track> \
  --output reel.mp4
```

**Narrated reel** (generate voice first, then compose):
```bash
# Step A: generate voiceover
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate_voice.py \
  --text "narration text" \
  --voice-ref <project_voice_ref> \
  --output narration.wav

# Step B: compose with narration
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate_slideshow.py \
  --images shot1.jpg shot2.jpg shot3.jpg \
  --music <track> \
  --narration narration.wav \
  --output reel.mp4
```

### Step 4: Review

Open the reel for the user to watch:
```bash
open -a "QuickTime Player" path/to/reel.mp4
```

Ask explicitly: "Does this look good? Approve to schedule, or describe what to change."

Do not proceed to scheduling without explicit approval.

### Step 5: Schedule or publish

Ask the user: schedule for a specific time, publish now, or save as draft?

**Schedule:**
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/schedule_instagram.py \
  --media reel.mp4 \
  --caption "caption text" \
  --schedule "2026-03-16T18:00:00" \
  --timezone "Europe/Berlin"
```

**Publish now:**
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/schedule_instagram.py \
  --media reel.mp4 \
  --caption "caption text" \
  --publish-now
```

## NSFW Content Workflow

For intimate or explicit content (members-only or PPV tiers). Requires Modal ComfyUI deployment and project-specific workflow templates.

**NSFW image:**
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate_nsfw_image.py \
  --prompt "scene description" \
  --output shot1.jpg \
  --face-ref <project_face_ref> \
  --workflow <project_workflow_template>
```

**NSFW video clip** (animate from a starting frame):
```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate_nsfw_video.py \
  --image starting_frame.jpg \
  --prompt "motion description" \
  --output clip.mp4 \
  --workflow <project_workflow_template>
```

SFW images generated by `generate_image.py` can be passed as the starting frame to `generate_nsfw_video.py` — Wan 2.2 NSFW will animate the transition.

---

## Important Rules

- Never post without explicit user approval
- Always show content to the user before scheduling
- Every reel is 3 shots composed into a video — no single-image posts
- Shots tell a mini story: wide/establish → mid/action → close/intimate
- All NSFW content must tie to a story beat defined in the project config — no random shots
