## Methodology

**App Store Featured** — Maximize chances of editorial featuring by Apple's App Store editors.

### What Apple Looks For

| Factor | Weight |
|--------|--------|
| Design quality | Very High |
| Apple technology adoption | Very High |
| Unique value proposition | High |
| Compelling story | High |
| Timeliness (seasonal, events) | High |
| Rating (≥ 4.5) | Medium |
| Stability (crash-free rate) | Medium |
| Accessibility | Medium |
| Privacy practices | Medium |

### Types of Featuring

| Type | Visibility | Difficulty |
|------|-----------|------------|
| App of the Day / Game of the Day | Highest (Today tab hero) | Very Hard |
| Today Tab Story | Very High (editorial article) | Hard |
| Apps We Love | High (curated collection) | Moderate |
| Category Feature | Medium | Moderate |
| Search Feature | Medium | Moderate |
| In-App Events | Low-Medium (self-serve) | Easy |

### Design Excellence Checklist

- [ ] Follows Apple Human Interface Guidelines
- [ ] Clean, consistent visual design
- [ ] Beautiful, distinctive app icon
- [ ] All 10 screenshot slots used
- [ ] App preview video uploaded
- [ ] Dark mode support
- [ ] iPad-optimized layout (if applicable)
- [ ] Dynamic Type / accessibility support

### Apple Technology Checklist (2025–2026 Priority)

Highest priority (Apple actively features apps using these):
- [ ] visionOS support
- [ ] Apple Intelligence integration
- [ ] App Intents / Siri Shortcuts

High priority:
- [ ] Live Activities (Dynamic Island)
- [ ] WidgetKit (home screen widgets)
- [ ] StoreKit 2
- [ ] App Clips

Good to have:
- [ ] SwiftUI (vs. UIKit)
- [ ] SharePlay
- [ ] Sign in with Apple

### Quality Signals Checklist

- [ ] Average rating ≥ 4.5 stars
- [ ] Updated within the last 3 months
- [ ] No critical bugs or crashes in recent version
- [ ] Launch time < 2 seconds
- [ ] Accurate privacy nutrition labels
- [ ] App Store Connect metadata complete and polished

### How to Submit for Consideration

**Self-Nomination Form:** `developer.apple.com/contact/app-store/promote/`

**Best Timing to Submit:**
- 2–4 weeks before a major update ships
- Around Apple events (WWDC, iPhone launch, holiday season)
- When adopting a new Apple technology for the first time
- When you have a compelling seasonal story

### Pitch Structure

**Subject:** [App Name] — [One Sentence Hook] — [Timing]

**Body (5 paragraphs):**
1. What the app does and who it's for (2 sentences)
2. What's new in the upcoming version (1–3 bullets)
3. Which Apple technologies are featured and how
4. The story angle (why is this app meaningful right now?)
5. Assets available + contact

**Include:**
- App Store link
- TestFlight link to the new version
- Press kit URL
- Recent ratings/review stats

### Featuring Readiness Score

Rate each area /10, target 8+ in all areas before pitching:

| Area | Score | Notes |
|------|-------|-------|
| Design Quality | /10 | HIG compliance, icon, screenshots |
| Apple Tech Adoption | /10 | Current-gen APIs used |
| Quality Signals | /10 | Ratings, stability, updates |
| Content & Story | /10 | Unique angle, compelling narrative |
| Rating | /10 | Based on current average |

**Total:** /50 — Submit when ≥ 40/50.

### In-App Events as Self-Serve Featuring

In-App Events appear in search results and the Today tab without needing editorial approval. See the `in-app-events` skill for detailed guidance on planning and executing events.
